1.0

Purpose

The /perception/detections topic carries all object detections from the YOLO model running on the OAK-D Lite's onboard NPU. Every detection regardless of class is published here. Downstream, the living_thing_filter bifurcates the stream into /perception/people and /perception/obstacles.

This topic is not directly consumed by safety. Safety listens to the filtered /perception/people topic; this raw stream exists for observability, debugging, and for any downstream consumer that wants custom filtering.

3.0

Message

Type: platform_msgs/msg/DetectionArray. Array of Detection records:

# platform_msgs/msg/Detection std_msgs/Header header string class # COCO class label, e.g. "person", "dog", "chair" float32 confidence # 0.0 to 1.0 geometry_msgs/Point position_rover_frame # 3D position in base_link frame, meters float32 distance_m # Euclidean distance from rover origin string source # "ground_camera" (iter 1) | "drone_camera" (future)

The header.stamp is the OAK-D capture time; frame_id is oak_color_camera_frame. Driver performs the coordinate transform from camera frame to base_link for position_rover_frame.

4.0

Semantics

Each array represents all detections from one inference cycle. An empty array means "no detections above confidence threshold," not "camera failed" (that would cause the topic to stop publishing).

Classes follow the COCO 80-class label set. The default confidence threshold is 0.4, tunable via confidence_threshold parameter. Below-threshold detections are dropped before publication to reduce bandwidth.

Position is derived from the stereo depth at the bounding-box centroid. Stereo baseline 7.5 cm (OAK-D Lite). Distance accuracy at 3 m is approximately ± 10 cm; at 10 m approximately ± 1 m.

5.0

Timing

Target 10 Hz; floor 8 Hz under nominal Pi load. Inference runs on OAK-D NPU; Pi overhead is post-processing and ROS serialization only.

6.0

QoS

RELIABLE, KEEP_LAST depth 2, VOLATILE. Small depth: an old detection is stale enough to be misleading; newest wins.

Requirements

    Detection publish rate shall be ≥ 8 Hz under nominal Pi load.
    verified by: TEST-PER-001
  • REQ-ICD-PER-001-02
    Detections below the configured confidence threshold shall not appear in the published array.
    verified by: TEST-PER-002
  • REQ-ICD-PER-001-03
    Position shall be reported in the base_link frame regardless of camera orientation.
    verified by: TEST-PER-003
