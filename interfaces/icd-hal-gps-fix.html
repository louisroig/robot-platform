/hal/gps/fix

GPS position fix from the HGLRC M80 receiver. Optional input for the global EKF in iteration 1; promoted to required in iteration 2 if RTK is integrated.

1.0

Purpose

Provides absolute (Earth-frame) position from the GPS receiver. In iteration 1, this is optional — the global EKF fuses SLAM pose with local EKF; GPS adds drift correction when available. For yards fully visible to the sky, GPS is beneficial; for tree-canopy yards, disable it.

Not used for safety. Not used for geofencing (geofence runs in fused-pose frame).

4.0

Semantics

When status.status < 0, the message is still published but covariance is set to infinity. Downstream consumers (EKF) reject such messages by covariance check.

Typical accuracy in a residential yard with clear sky: ± 2.5 m CEP on standalone GPS; ± 1 m with SBAS. Iteration 2 targets RTK (± 2 cm) when adding a base station.

5.0

Timing

5 Hz when fixed; 1 Hz (with status.status = NO_FIX) when searching. Driver bridges u-blox UBX at 9600 baud.
