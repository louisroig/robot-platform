1.0

Purpose

The /hal/cmd_vel_raw topic carries the arbitrated velocity command from twist_mux — the highest-priority currently-active velocity source, selected from teleop, autonomous, and recovery inputs. It is the single input consumed by safety_monitor for safety gating.

The "raw" suffix denotes that the command has not yet passed through safety gating; it reflects operator/mission intent only.

3.0

Message Definition

Type: geometry_msgs/msg/Twist. Field semantics identical to ICD-HAL-002 §3, with the following differences:

  • Field values carry operator / mission intent, not gated output. Bounds (0.7 m/s, 1.5 rad/s) still apply; twist_mux priorities already limit inputs.
  • Non-finite values should not occur but are not enforced here; enforcement is at the safety gate.
4.0

Semantics

A message on this topic represents "what the system wants the rover to do, absent safety considerations." Whether the rover actually does it depends on safety_monitor's current state.

Priority arbitration rules (from twist_mux config):

  • teleop (100) wins whenever a Twist is received on /hal/cmd_vel_teleop within the last 500 ms.
  • auto (50) takes over when teleop is silent (iteration 2+). Iteration 1: stubbed.
  • recovery (10) takes over when both above are silent. Iteration 1: unused.
  • When no source is active, twist_mux ceases publishing — the safety monitor then sees staleness and begins publishing zero.

Fault Behavior
