drone_mission state machine

The drone's mission-cycle state, owned by drone_mission_coordinator. Nine sequential states + ABORTING.

1.0

Purpose

Formal specification of the state machine inside drone_mission_coordinator that drives a mapping mission from goal-accept through land. The mission-action-interface in ICD-MAP-003 observes this state via feedback.

2.0

Diagram

The diagram shows the in-flight portion of the cycle. The post-land states IMAGE_TRANSFER and GEOREF follow LANDED on the success path; see §3 and §4 for the full enumeration.

5.0

Abort Semantics

  • ABORTING is in-flight-only. It always commands immediate LAND via MAV_CMD_NAV_LAND. The drone descends at its current lat/lon.
  • No attempt to return to takeoff point is made in iteration 1 (would require RTL mode transition and extra seconds). The safety argument is: "already in the air, already over your yard — coming straight down is fine."
  • Once ABORTING is entered, subsequent cancel / safety events are no-ops (we're already aborting).
  • From the abort path no IMAGE_TRANSFER is attempted — there's nothing meaningful to retrieve, and the operator wants closure quickly.
  • The action result is populated on entry to DISARMED — either via GEOREF (success) or directly from LANDED on the abort path (failure) or from IMAGE_TRANSFER on transfer failure (failure with drone already safe).
