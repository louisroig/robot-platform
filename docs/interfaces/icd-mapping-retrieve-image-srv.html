/mapping/retrieve_latest_image

Pulls the most recent capture from the drone's XIAO ESP32-S3 to rover disk and returns the local image path.

1.0

Purpose

The /mapping/retrieve_latest_image service is the explicit hand-off between the drone-side capture (XIAO HTTP, see ICD-XIAO-001) and the rover-side persistence + georeferencing pipeline. It is called by drone_mission_coordinator after the IMAGE_TRANSFER state begins, and returns the absolute path of the saved file on rover disk.

Splitting capture and retrieval into separate steps — instead of letting the XIAO push to the rover — keeps the XIAO firmware tiny (HTTP server only) and keeps retry/error semantics on the rover side where Python and the ROS log are easier to work with.

Interface Definition

Service type: platform_msgs/srv/RetrieveLatestImage.

# Request string mission_id # namespaces the saved file: /var/lib/platform/images/<mission_id>/capture.jpg --- # Response bool success string image_path # absolute path on rover disk on success; empty string on failure string message # human-readable detail; empty on success, error explanation on failure

Example success response:

{ "success": true, "image_path": "/var/lib/platform/images/2026-05-03T19-21-44Z/capture.jpg", "message": "" }

Example failure response (XIAO unreachable after retries):

{ "success": false, "image_path": "", "message": "xiao unreachable: 3 attempts to http://xiao-bridge.local/latest.jpg timed out after 10s each" }
4.0

Semantics

On invocation, image_retriever:

  1. Validates mission_id is non-empty and contains only filename-safe characters (alphanumeric, dash, underscore, ISO-8601 colons replaced with dashes). Empty or unsafe → SUCCESS=FALSE with reason.
  2. Constructs the destination directory <image_dir>/<mission_id>/, creating it if absent.
  3. Issues GET http://<xiao_host>/latest.jpg per ICD-XIAO-001 §3, streaming the body to capture.jpg in the destination directory.
  4. On HTTP error or timeout, retries per SRS-MAP-002-F02 (3 attempts, 1 s / 2 s / 4 s exponential backoff). Failures within retry budget are not visible to the caller.
  5. On final success, returns the absolute path. The file is fsync'd before the response is sent, so a downstream call to /mapping/georeference is guaranteed to find a complete, durable file.
  6. On final failure (retries exhausted or unrecoverable error), returns SUCCESS=FALSE with a single-line description suitable for surfacing to the operator app.

The service is not idempotent in the strict sense — each call performs a fresh HTTP fetch and overwrites the destination file. Calling twice on the same mission_id yields two equivalent files (or two failures); there is no caching layer.

6.0

QoS

ROS 2 service default QoS. Reliability RELIABLE (services require it). The HTTP transport underneath is best-effort TCP per ICD-XIAO-001; service-level reliability is provided by image_retriever's retry loop, not by the underlying socket.

Concurrent callsEach call gets its own HTTP attempt. The destination filename is fully determined by mission_id, so calls with distinct ids cannot collide; calls with the same id race on the file (last writer wins). Coordinator does not issue concurrent calls per mission, so this is theoretical.

Requirements

  • REQ-ICD-MAP-004-01
    On SUCCESS, the file at image_path shall exist and be a complete, fsync'd JPEG before the response is returned.
    verified by: TEST-MAP-040
  • REQ-ICD-MAP-004-02
    On retry-exhaustion failure, the partial download (if any) shall be removed from disk before the response is returned, so a subsequent SUCCESS call cannot accidentally serve a half-file.
    verified by: TEST-MAP-041
  • REQ-ICD-MAP-004-03
    Service callback latency shall be bounded by §5: ≤ 37 s worst case, typically ≤ 5 s.
    verified by: TEST-MAP-040 · TEST-MAP-041

Open Issues

  1. OPEN-01 — Multi-image support: iter 1 captures one image per mission. When mosaicking lands (iter 2+), this service likely takes a list-of-image-ids in the request and returns a list of paths. Defer signature change until the consumer is real.
