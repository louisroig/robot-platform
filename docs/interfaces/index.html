About the ICDs

Every topic, service, and action in the system has an Interface Control Document. Each ICD follows the same shape: purpose, participants (publisher/subscriber cards with cross-links to node SRSes), message definition, semantics, timing & rate, QoS profile, behavior under fault, requirements, verification, open issues, and revision history.

The ICDs are the most heavily cross-linked documents in the corpus. If you are tracking a requirement or debugging a behavior, start here. Every node SRS lists its consumed and produced interfaces with inbound links; every requirement that bears on an interface is echoed in the relevant ICD.

Safety-critical interfaces

Three interfaces carry safety-critical semantics: /hal/cmd_vel_safe, /safety/state, and /geofence/state. Changes to these require explicit review against the safety requirements in SR-*.
