drone_mission_coordinator

Orchestrates the drone's complete mapping mission cycle: arm, takeoff, capture, land.

1.0

Purpose

Implements the action server behind /mapping/run_mission. Commands the drone through a fixed sequence (arm → takeoff → climb → capture → descent → land) via MAVROS. Owns a state machine and feeds progress to the action feedback channel.

2.0

Scope

In: action server, MAVROS command sequencing, capture trigger to XIAO, image-retrieval kickoff, state feedback. Out: physical flight control (lives in ArduCopter); georeferencing (lives in georeferencer).

3.0

Interfaces

Action server: /mapping/run_mission. MAVROS clients (services and topics): /mavros/cmd/arming, /mavros/cmd/takeoff, /mavros/cmd/land, /mavros/state, /mavros/global_position/global. Publisher: /diagnostics.

4.0

Functional

  • SRS-MAP-001-F01
    Execute the mission sequence within the timing envelope: accept within 100 ms; complete within 240 s for a 30 m altitude capture.
    verified by: TEST-MAP-020
  • SRS-MAP-001-F02
    Emit progress feedback at ≥ 1 Hz throughout execution.
    verified by: TEST-MAP-021
  • SRS-MAP-001-F03
    On cancel, goal abort, or safety veto, transition to ABORTING and command immediate land.
    verified by: TEST-MAP-022
5.0

Performance

CPU < 5%. Feedback rate ≥ 1 Hz. Action accept latency ≤ 100 ms.

6.0

Safety

  • SRS-MAP-001-S01
    Pre-arm check shall reject missions when /safety/state != OK.
    maps to: SR-008
  • SRS-MAP-001-S02
    Loss of MAVROS heartbeat (/mavros/state not received > 2 s) shall trigger abort.
    maps to: SR-006
7.0

State Machine

DISARMED → ARMED → TAKEOFF → CLIMBING → CAPTURING → DESCENDING → LANDED → IMAGE_TRANSFER → GEOREF → DISARMED on the success path; ABORTING short-circuits any in-flight state to LANDED → DISARMED with no IMAGE_TRANSFER. Action result is emitted on entry to DISARMED. See SM-DRN-001.

8.0

Dependencies

rclpy, mavros, platform_msgs, image_retriever service contract.

9.0

Configuration

ParameterDefault
default_altitude_m30.0
climb_timeout_s45
capture_hover_s3
abort_on_low_battery_pct20.0

Battery threshold layering. abort_on_low_battery_pct must trip before the ArduCopter battery failsafe (FW-ACP §3, BATT_LOW_VOLT) so the coordinator owns the abort decision and emits a clean action result. If the FC failsafe wins the race, the coordinator sees the drone descending without having issued the command and the action result is best-effort. To preserve coordinator-first ordering on a 3S 550 mAh pack, abort_on_low_battery_pct should leave ≥ 5% margin over the FC trigger under typical voltage sag — verify on bench before flight tuning.

10.0

Failure Modes

Arm rejection, GPS loss, altitude timeout, low battery mid-mission, capture failure — all lead to ABORTING with land. Detailed in SM-DRN-001.

11.0

Verification
