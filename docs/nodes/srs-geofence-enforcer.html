geofence_enforcer

Computes rover position relative to the active zone polygon and classifies it as INSIDE, WARNING, or OUTSIDE.

1.0

Purpose

Subscribes to /mapping/current_zone and /localization/pose. Computes signed point-in-polygon distance. Publishes /geofence/state with classification and distance. Applies hysteresis to avoid state chatter at the boundary.

2.0

Scope

In: signed distance via ray-casting or Shapely; hysteresis-managed state transitions. Out: zone activation (belongs to boundary_ingester); coordinate conversion (done upstream).

3.0

Interfaces

Subscribers: /mapping/current_zone, /localization/pose. Publishers: /geofence/state, /diagnostics.

4.0

Functional

  • SRS-SAF-002-F01
    For each pose update, compute signed distance to the nearest edge of the outer polygon (negative if outside) minus the signed distance to the nearest exclusion polygon edge.
    verified by: TEST-SAF-032
  • SRS-SAF-002-F02
    Classify INSIDE (distance ≥ warning_threshold), WARNING (0 < distance < threshold), OUTSIDE (distance ≤ 0).
    verified by: TEST-SAF-030
  • SRS-SAF-002-F03
    Apply 0.25 m hysteresis on threshold crossings.
    verified by: TEST-SAF-031
5.0

Performance

Publication rate 10 Hz ± 1 Hz. Polygon-distance compute time ≤ 5 ms per cycle for polygons up to 200 vertices.

6.0

Safety

  • SRS-SAF-002-S01
    Stale pose input shall cause the node to halt publication rather than publish last-known; downstream safety_monitor handles via staleness logic.
    maps to: SR-006
7.0

State Machine

Three states with hysteresis. See SM-GEO-001.

8.0

Dependencies

rclpy, Shapely, platform_msgs.

9.0

Configuration

ParameterDefault
warning_threshold_m1.0
hysteresis_m0.25
publication_rate_hz10
10.0

Failure Modes

Stale pose: halt publication. No zone loaded: emit INSIDE + infinite distance (geofencing inactive). Degenerate polygon: should not occur (rejected upstream); if somehow encountered, log and halt.

11.0

Verification

TEST-SAF-030/031/032 cross-referenced from ICD-SAF-003.
