RTAB-Map Configuration

Visual SLAM via RTAB-Map using the OAK-D Lite FF stereo pair. Deployment profile and tuning highlights.

Document ID
INT-RTAB
Revision
0.1 · draft
Status
Iteration 1
1.0

Purpose

Visual SLAM for the ground rover. RTAB-Map consumes the OAK-D Lite FF's stereo image streams and the IMU, produces a pose estimate and a 3D feature map, and publishes pose via /rtabmap/odom — which the global EKF fuses with /odometry/filtered.

2.0

Pipeline

  • Input. Stereo left+right color from OAK-D Lite FF at 720p 15 fps; IMU pre-integrated via /hal/imu/data.
  • Front-end. GFTT+BRIEF features (fast on CPU; sufficient for outdoor grass-and-foliage scenes).
  • Back-end. g2o optimization on loop closures.
  • Output. /rtabmap/odom (pose in map frame), /rtabmap/grid_map (unused in iter 1).

Known Limitations

  • Featureless surfaces (large uniform lawns) degrade tracking. Mitigation: keep odom+IMU fused at high weight when visual features are sparse.
  • Dynamic scenes (moving people, pets) create spurious loop closures. Partial mitigation via RTAB-Map's dynamic-object filtering; full mitigation out of scope for iteration 1.
  • CPU: ~30-40% of one Pi 5 core at 2 Hz detection. Acceptable margin.
