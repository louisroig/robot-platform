motor_driver

Open-loop dual-PWM control of two IBT-2 (BTS7960B) H-bridge modules for track drive. Converts Twist commands into per-track signed duty on RPWM/LPWM channel pairs. No encoder feedback; no battery telemetry.

1.0

Purpose

The motor_driver is the vendor-boundary node for two IBT-2 (BTS7960B) 43 A H-bridge modules, controlling the left and right aluminum tracks. Upstream it exposes ROS-standard Twist commands; downstream it drives each H-bridge via a pair of hardware PWM channels (RPWM for forward, LPWM for reverse) on Pi 5 GPIO, per HW-PI5-001. This is open-loop control: no encoder feedback. Above it, the system treats motor control as generic; below it, the driver handles all GPIO-level specifics.

2.0

Scope

In scope. Dual-PWM (RPWM/LPWM) output to the IBT-2 pair; Twist-to-per-track velocity kinematics; command-timeout safe-halt; basic thermal shutdown detection via stalled-motion cross-check with IMU.

Out of scope. Encoder reading and odometry (deferred to M5 EKF). Battery voltage monitoring (deferred, requires external ADC). Nav2 costmap, trajectory following, slip compensation.

Functional Requirements

  • SRS-HAL-001-F01
    Convert every received Twist into per-track velocities via skid-steer kinematics using configured track width.
    verified by: TEST-HAL-001
  • SRS-HAL-001-F02
    Clip commanded velocities to configured max_linear_vel and max_angular_vel with sign preservation.
    verified by: TEST-HAL-012
  • SRS-HAL-001-F03
    Drive each IBT-2 with two hardware PWM channels (RPWM, LPWM) at configured frequency (default 2 kHz). At any instant at most one of the two channels per track shall be non-zero, yielding a signed duty on [−1, 1]: RPWM = duty when commanding forward, LPWM = |duty| when commanding reverse, both zero for coast/regen-brake. Simultaneous non-zero RPWM and LPWM is prohibited (H-bridge short).
    verified by: TEST-HAL-002
  • SRS-HAL-001-F04
    Halt motor commands if no /hal/cmd_vel_safe message has been received within cmd_vel_timeout_ms (default 500 ms).
    verified by: TEST-HAL-009
  • SRS-HAL-001-F05
    Reject Twist messages containing NaN or Inf in any field; emit diagnostic; hold last-known-good command or halt per policy.
    verified by: TEST-HAL-011

Performance Requirements

  • SRS-HAL-001-P01
    GPIO write cycle per control frame ≤ 5 ms (4 hardware PWM channels).
    verified by: TEST-HAL-013
  • SRS-HAL-001-P02
    CPU usage < 3% on Pi 5.
    verified by: TEST-HAL-014
6.0

Safety Requirements

  • SRS-HAL-001-S01
    Command timeout (F04) is the subscriber-side component of the two-layer safety: even if safety_monitor fails to publish zero, stale commands themselves cause halt.
    maps to: SR-005
  • SRS-HAL-001-S02
    On startup, motors shall be held at zero velocity until first valid /hal/cmd_vel_safe is received.
    maps to: SR-008

Open Issues

  1. OPEN-01 — PWM duty-to-velocity mapping is empirical (calibrated for track slip at various speeds); calibration wizard in iteration 2.
  2. OPEN-02 — IMU-based stall detection is a coarse health check; encoder-less motor current sensing (via shunt resistor on H-bridge) deferred to iteration 2.
  3. OPEN-03 — Closed (rev 0.4). BCM pin assignments frozen in HW-PI5-001 §3; dual-PWM channels on GPIO 12/13 (left track RPWM/LPWM) and 18/19 (right track RPWM/LPWM).
