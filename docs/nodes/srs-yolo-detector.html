yolo_detector

Runs YOLO object detection on the OAK-D Lite FF's onboard NPU and publishes spatial detections for downstream consumers.

1.0

Purpose

Loads a YOLOv8n model onto the OAK-D Lite FF's VPU, runs inference on the color image stream, fuses 2D bounding-box centroids with the camera's onboard stereo depth to estimate 3D position in the rover's base_link frame, and publishes a DetectionArray.

2.0

Scope

In: DepthAI pipeline setup, model load, inference cadence, spatial centroid extraction, frame transform to base_link, publication. Out: semantic segmentation, multi-model voting, tracking (iter 2+).

3.0

Interfaces

Publishers: /perception/detections (target 10 Hz), /diagnostics. Subscribers: none (camera stream is internal to DepthAI).

4.0

Functional

  • SRS-PER-001-F01
    Run YOLOv8n (5-class subset configured for relevant outdoor objects) on the OAK-D NPU at ≥ 10 fps.
    verified by: TEST-PER-001
  • SRS-PER-001-F02
    For each detection above confidence_threshold, compute 3D position via the OAK-D's spatial-detection feature.
    verified by: TEST-PER-003
  • SRS-PER-001-F03
    Transform detection coordinates from camera frame to base_link using the static camera pose parameters.
    verified by: TEST-PER-003
5.0

Performance

  • SRS-PER-001-P01
    Inference rate ≥ 10 fps; floor 8 fps under thermal throttling.
    verified by: TEST-PER-001
  • SRS-PER-001-P02
    Pi-side CPU < 15% (inference runs on VPU).
    verified by: TEST-PER-005
  • SRS-PER-001-P03
    Frame-to-publish latency < 150 ms p95.
    verified by: TEST-PER-006
6.0

Safety

Although not itself safety-critical, the node's output feeds safety-critical filtering. If OAK-D becomes silent, downstream staleness triggers ESTOP via safety_monitor. Node shall not emit fabricated detections when the camera is unavailable.

7.0

Dependencies

  • depthai SDK.
  • depthai_ros_driver (off-the-shelf) for model install helpers.
  • platform_msgs.

9.0

Failure Modes

  • USB disconnect: DepthAI exception; node exits; systemd restart.
  • Model blob missing: node refuses to start with a specific log line pointing to the blob path.
  • Overheating: OAK-D throttles inference rate; node logs warning; downstream reacts to staleness.
10.0

Verification

TEST-PER-001 (rate), TEST-PER-002 (confidence filtering), TEST-PER-003 (frame transform), TEST-PER-005 (CPU), TEST-PER-006 (latency).

11.0

Open Issues

  1. OPEN-01 — Model selection. YOLOv8n is initial; YOLOv10n or a smaller dedicated "person+pet" model may give better latency / accuracy. Benchmark pending.
  2. OPEN-02 — Temporal filtering (a cheap tracker for stability) is iteration 2.
  3. OPEN-03 — Night / low-light. OAK-D has IR but no IR illuminator on iteration 1. Low-light operation is out of envelope.
