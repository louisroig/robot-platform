xiao_bridge

Firmware for the XIAO ESP32-S3 Sense — Wi-Fi MAVLink bridge and OV2640 camera capture for the drone.

Document ID
FW-XIAO
Revision
0.3 · draft
Target
XIAO ESP32-S3 Sense
Framework
Arduino-ESP32
1.0

Purpose

The XIAO ESP32-S3 Sense sits onboard the drone. It has two jobs: (1) transparently bridge MAVLink between the FC's TELEM2 UART and the Pi's mavros TCP client over Wi-Fi; (2) capture a still image on command via the onboard OV2640 camera and make it available over HTTP.

2.0

Hardware

  • XIAO ESP32-S3 Sense (ESP32-S3R8, 8 MB flash, 8 MB PSRAM, OV2640 camera module).
  • UART0 connected to FC TELEM2 at 57600 baud, 8N1.
  • Micro-SD slot used as optional image cache.
3.0

Role 1 — MAVLink Bridge

  • On boot, XIAO connects to the configured Wi-Fi (station mode).
  • Opens TCP listener on port 5760.
  • Advertises its hostname via mDNS as xiao-bridge.local; mavros on the Pi connects to that name (no static-IP coordination required, no router-side DHCP reservation needed).
  • When a client connects, bidirectionally forwards bytes between UART and TCP with minimal buffering (≤ 5 ms one-way latency).
  • Supports exactly one client at a time (mavros on Pi). A second connection attempt while a client is already connected is rejected immediately (TCP RST + log line); the existing client is not displaced.
4.0

Role 2 — Image Capture

  • HTTP server on port 80.
  • GET /capture — triggers a capture; stores to a rolling buffer; responds with the JPEG.
  • GET /latest.jpg — returns the most recent captured frame without re-triggering.
  • Capture parameters: 1600×1200 (UXGA), JPEG quality 12 (ESP32 camera library convention: range 0–63, lower = better quality), auto-exposure default.
  • Trigger latency (GET → image in buffer) ≤ 500 ms.

The wire-level contract — exact response codes, headers, concurrency behavior, and verification — is normative in ICD-XIAO-001. This section is the firmware implementer's view; ICD-XIAO-001 is the consumer's view. A change to either requires a coordinated rev of the other.

5.0

Configuration

Wi-Fi credentials and static config in a compile-time header include/config.h. Runtime re-config not supported in iteration 1.

6.0

Failure Modes

  • Wi-Fi disassociation: auto-reconnect with backoff.
  • Camera timeout: return 503 on /capture; keep bridge alive.
  • Watchdog timeout on main loop: device reset (ESP-IDF watchdog).
7.0
