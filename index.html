SPECIFICATION CORPUS · ITERATION 1

Ground-Air Autonomous
Platform

A coordinated rover and drone system for supervised outdoor autonomy. First application: residential yard automation.

PREPARED BY LOUIS ROIG & CLAUDE OPUS 4.7  ·  REV 0.1 · DRAFT  ·  APRIL 2026

This specification corpus defines the first iteration of a ground-air autonomous platform — a coordinated rover and drone system designed as a generalizable foundation for remote-operated outdoor robotics applications. The initial instantiation targets residential yard automation, specifically a drone-assisted boundary-mapping workflow feeding a tracked ground unit operated under human supervision. The platform architecture, interfaces, and safety framework defined herein are explicitly generic; mower-specific behaviors are isolated in the application layer. Subsequent iterations will introduce autonomous coverage, a mower tool payload, and commercial-grade safety architecture.

Guiding Principles

Principle 01
Platform first, application second

All generic code lives in /platform/. Mower-specific logic is confined to /applications/mower/. The first application exists to prove the platform.

Principle 02
Safety by layering

Hardware e-stop, safety MCU (iteration 2+), and the ROS-level safety monitor are three independent layers. No single failure defeats safety.

Principle 03
HAL is the vendor boundary

Above the HAL = vendor-neutral. Below the HAL = vendor-specific. Swapping the motor controller or flight controller affects only the HAL.

Principle 04
Spec before code

Every node begins with an SRS. Every interface begins with an ICD. This corpus is the artifact that embodies the practice.
