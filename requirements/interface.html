About this document

Each ICD in the interfaces catalog contains its own local requirements (REQ-ICD-XXX-NN). This document captures only the cross-cutting interface requirements that apply to the system as a whole — architectural constraints on how interfaces are named, versioned, and evolved.

2.0

Requirements

  • IR-001
    All ROS 2 topics, services, and actions in the platform shall conform to the corresponding ICD in /interfaces/, including QoS profile, message type, and semantic rules.
    verified by: code review + TEST-QOS-* runtime checks
  • IR-002
    All safety-critical topics shall have exactly one authorized publisher; additional publishers shall be detected at runtime.
    maps to: SR-009 · allocated to: system_monitor · verified by: TEST-SAF-016
  • IR-003
    Platform message types (platform_msgs) shall use generic names (tool, terrain, area, zone) and shall not reference application-specific concepts.
    maps to: ARCH-005 · verified by: documentation lint + code review
  • IR-004
    Every external interface (REST, WebSocket) shall require authentication; no endpoint shall expose rover state or command capability without a valid bearer token.
    allocated to: mission_api, telemetry_publisher · verified by: TEST-API-001, TEST-API-012
  • IR-005
    ICDs shall be versioned. Breaking changes to a published interface require a revision bump and a documented migration path.
    Guards against undetected downstream breakage when interface semantics change.
    verified by: revision history in each ICD; code review
  • IR-006
    All topic timing properties (rate, deadline, lifespan) shall match the published ICD within stated tolerances at runtime.
    allocated to: individual publishers · verified by: per-ICD rate tests
