Purpose

Formal specification of the state machine in geofence_enforcer — a three-state classifier with hysteresis over the continuous signed distance to the active polygon.

4.0

Transitions

Let d = signed distance to nearest edge (positive inside, negative outside). Let T = warning_threshold_m (1.0), H = hysteresis_m (0.25).

FromToTrigger
INSIDEWARNINGd < T − H (i.e. d < 0.75)
WARNINGINSIDEd > T + H (i.e. d > 1.25)
WARNINGOUTSIDEd < −H (i.e. d < −0.25)
OUTSIDEWARNINGd > +H (i.e. d > 0.25)
INSIDEOUTSIDEonly via WARNING (no direct jump allowed)

Direct INSIDE→OUTSIDE is not permitted; the machine must pass through WARNING. This prevents chatter when distance estimates jump discretely due to SLAM pose corrections.

5.0

Hysteresis

Hysteresis of 0.25 m prevents rapid state cycling when the rover lingers near a threshold. The symmetric "band" around each threshold means:

  • To enter a state, cross its threshold by at least H in the triggering direction.
  • To exit the same state back the way you came, cross the threshold by at least H in the opposite direction.

This works with the 10 Hz publication rate: a rover-sized object moving at 0.5 m/s covers 0.5 m / 0.1 s = 50 mm per tick — comfortable relative to the 0.25 m hysteresis.

6.0

Open Issues

  1. OPEN-01 — Disallowing INSIDE→OUTSIDE direct is conservative; a sudden pose-correction jump may leave the rover momentarily in OUTSIDE with no intermediate sample. In that case the first visible state is OUTSIDE — safe, but logs look odd.
  2. OPEN-02 — Per-side hysteresis width (e.g., tighter for exclusion polygon edges) deferred to iteration 2.
