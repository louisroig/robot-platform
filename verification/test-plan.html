1.0

Purpose

Umbrella document for iteration-1 verification. Describes the test strategy, the test methods and where each applies, the tooling we use, how CI runs them, what coverage we target, and the entry/exit criteria for progressing between test phases. Individual test protocols live in their own documents; this document is the frame they plug into.

2.0

Strategy

Iteration-1 verification is pragmatic, not certification-oriented. Three principles:

  1. Safety first. Safety-critical requirements get the highest-fidelity verification available: integration tests with real DDS, fault injection, and the M6 field test. Safety correctness is the bar; there is no "good enough" for SR-001 through SR-010.
  2. Unit-first for non-safety. Pure-function components (georeferencing math, polygon validation, filter routing) are unit-tested with mocks. Cheap and fast.
  3. Field test is the final arbiter. The M6 field test is the single pass/fail gate for iteration-1 closure. Until it runs cleanly five times, iteration 1 is not done.

4.0

Tooling

  • pytest for Python unit and integration tests.
  • Google Test (gtest) for C++ unit tests (safety_monitor).
  • launch_testing (ROS 2) for orchestrating multi-node integration tests with inline assertions.
  • ros2 topic hz / rostopic echo as dev-bench utilities.
  • rosbag2 for record/replay in integration and chaos tests.
  • pytest-timeout, pytest-asyncio as support plugins.
5.0

CI / CD

  • GitHub Actions runs on every PR and main-branch commit.
  • Matrix covers: Ubuntu 24.04 (ROS 2 Humble arm64 container), Python 3.12, gcc 13.
  • Unit + integration tests run in CI. System (HIL) and field tests are manual.
  • Failing test blocks merge.
  • Static checks: ament_cpplint, ament_flake8, mypy on Python, and the custom platform/application-boundary lint (ARCH-005 §3).

7.0

Entry / Exit Criteria

7.1 Entry to HIL (System) testing
  • All unit and integration tests passing for 1 week.
  • Safety-monitor SRS fully implemented.
  • Bench hardware assembled and calibrated (ticks-per-meter, IMU).
7.2 Entry to field testing
  • All HIL tests passing for 2 days of continuous operation (with simulated commands).
  • E-stop button installed and verified by physical inspection.
  • Operator qualified on mobile app and understands abort procedures.
7.3 Exit — Iteration 1 complete
  • VRF-FIELD passes 5 consecutive trials.
  • No known safety-critical bugs open.
  • All SR-* and PR-* requirements verified.
  • Traceability matrix shows no unverified rows.
