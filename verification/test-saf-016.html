Rogue publisher detection

Integration test: verify that an unauthorized second publisher on /hal/cmd_vel_safe is detected and forces safety to ESTOP.

Test ID
TEST-SAF-016
Method
Integration
Automation
CI (launch_testing)
1.0

Requirements verified

  • SR-009 — detect and reject unauthorized publishers on safety-critical topics.
  • REQ-ICD-002-05 — only safety_monitor shall publish.
  • IR-002 — exactly one authorized publisher.
2.0

Method

Integration test. Full safety stack launched. A test harness spawns a second, unauthorized publisher on /hal/cmd_vel_safe. The test verifies that system_monitor detects the duplicate publisher within 5 s and transitions /safety/state to ESTOP with reason "rogue_publisher".

3.0

Preconditions

  • safety_monitor running, publishing /hal/cmd_vel_safe.
  • system_monitor running with default publisher_count_poll_hz = 1 Hz.
  • /safety/state is initially OK.
4.0

Procedure

  1. Confirm initial state: /safety/state = OK for at least 2 s.
  2. Start a second publisher node (rogue_publisher.py) that publishes Twist at 10 Hz on /hal/cmd_vel_safe with non-zero velocity.
  3. Record timestamp t_rogue when the rogue publisher registers.
  4. Monitor /safety/state; find the first message with state == ESTOP (2) after t_rogue. Record t_estop.
  5. Verify reasons[] contains a string starting with "rogue_publisher".
  6. Compute latency = t_estop − t_rogue.
  7. Stop the rogue publisher. Wait 10 s. Verify /safety/state does NOT auto-clear (this is a latched condition).
  8. Call /safety/reset; verify it returns success=TRUE after the rogue has been gone for enough samples; verify state transitions to OK.
5.0

Expected results

  • Latency ≤ 5 s (accounting for 1 Hz poll + 2-sample confirmation window).
  • reasons[] contains "rogue_publisher:/hal/cmd_vel_safe".
  • State stays at ESTOP even after rogue stops publishing.
  • /safety/reset succeeds only when rogue has been absent for at least 2 polling cycles.
6.0

Pass / fail

Pass: All of: latency ≤ 5 s, correct reason string, latched until reset, reset works once condition cleared.

Fail: Latency > 5 s, wrong reason, state auto-clears without reset, or reset succeeds while rogue still present.

7.0
