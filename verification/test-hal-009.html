Motor driver command timeout

Integration test: verify motors halt within 500 ms of loss of /hal/cmd_vel_safe. Second independent safety layer.

Test ID
TEST-HAL-009
Method
Integration
Automation
CI (launch_testing)
1.0

Requirements verified

  • SR-005 — safety-critical components operate independently.
  • REQ-ICD-002-03 — subscriber halts within 500 ms of last message.
  • SRS-HAL-001-F04 — command timeout.
2.0

Method

Integration test. roboclaw_driver runs with a mock serial adapter that records every sent command to an in-memory log. A test harness publishes cmd_vel_safe, then stops publishing. The test verifies that the next motor command sent to the mocked serial is zero within 500 ms of the last published cmd_vel_safe.

3.0

Preconditions

  • roboclaw_driver launched with cmd_vel_timeout_ms = 500.
  • Mock serial attached; commands logged with microsecond timestamp.
  • No other publisher on /hal/cmd_vel_safe.
4.0

Procedure

  1. Publish Twist(linear.x=0.3, angular.z=0) on /hal/cmd_vel_safe at 50 Hz.
  2. Verify the mock serial log shows non-zero motor commands.
  3. Stop publishing at time t_stop.
  4. Scan the mock serial log for the first motor command with both tracks at zero after t_stop. Record t_zero.
  5. Compute latency = t_zero − t_stop.
  6. Repeat steps 1–5 ten times.
5.0

Expected results

  • Every trial's latency ≤ 500 ms.
  • p95 < 550 ms (accounting for the 500 ms threshold + ~50 ms margin for serial hop).
  • No non-zero motor command is issued after t_zero.
6.0

Pass / fail

Pass: All 10 trials latency ≤ 600 ms (100 ms margin beyond spec); no late non-zero commands.

Fail: Any trial > 600 ms, or any non-zero command issued beyond t_zero + 100 ms.

7.0

Fixtures

test/fixtures/mock_roboclaw_serial.py — implements the RoboClaw binary protocol at serial-packet level and logs commands.

8.0
