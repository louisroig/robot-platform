Verification strategy

Verification for iteration 1 is pragmatic: unit tests for pure-function behavior, integration tests for interactions between nodes, and a single field test for end-to-end operator-in-the-loop evidence. Formal analysis, fault injection at scale, and certification-oriented methods come in later iterations.

The test plan is the umbrella document. The three example test protocols show how a requirement is traced into a concrete, automatable test. The M6 field test is the pass/fail gate for iteration 1.

MethodScopeWhere it runs
UnitSingle class / function, mocked dependenciespytest / gtest in CI
IntegrationMultiple nodes via launch_testing, real DDS middlewarelaunch_testing in CI
