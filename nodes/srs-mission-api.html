mission_api

FastAPI service bridging the mobile app to ROS 2. Hosts the REST surface and owns the bearer-token store.

1.0

Purpose

Serves the REST surface defined in ICD-API-001. Translates between HTTP requests and ROS 2 topics/services/actions. Enforces authentication. Validates incoming data (polygon well-formedness, Twist bounds).

2.0

Scope

In: HTTP handlers, request validation, token store, ROS 2 bridge layer, response shaping. Out: WebSocket telemetry (lives in telemetry_publisher); mobile app UI.

3.0

Interfaces

External: HTTPS on TCP 8443. Full endpoint list: ICD-API-001. Internal ROS 2: subscribes to /safety/state, /localization/pose, /hal/battery; publishes /hal/cmd_vel_teleop; calls /safety/reset, /mapping/load_zone; invokes /mapping/run_mission action.

4.0

Functional

  • SRS-API-001-F01
    Expose every endpoint defined in ICD-API-001 §3 with the documented request/response contract.
    verified by: TEST-API-001..TEST-API-003
  • SRS-API-001-F02
    Reject any request without a valid bearer token with HTTP 401 before any side effect.
    verified by: TEST-API-001
  • SRS-API-001-F03
    Before forwarding teleop commands to ROS, verify safety state = OK; return 503 otherwise.
    verified by: TEST-API-003
5.0

Performance

p95 latency: /rover/state ≤ 50 ms; /rover/teleop ≤ 30 ms; /missions/map ≤ 500 ms to accept; image fetch dominated by network.

6.0

Safety

  • SRS-API-001-S01
    A successful /rover/estop call shall result in safety_monitor observing the request within 200 ms.
    verified by: TEST-API-002
7.0

Security

  • TLS 1.2+ required on all endpoints.
  • Bearer tokens stored hashed (Argon2id).
  • Per-request audit trail written to /var/log/platform/api-access.log.
8.0

Dependencies

FastAPI, uvicorn, rclpy, platform_msgs, passlib (Argon2), pydantic.

9.0

10.0

Failure Modes

ROS 2 node unavailable: endpoints return 503 with diagnostic message. TLS cert missing: refuse to start. Token store corrupt: refuse to start (log instructions).

11.0

Verification
