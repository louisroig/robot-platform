Single authority for motion and tool engagement. Aggregates safety-relevant signals from perception, sensors, and hardware into a unified safety state. Gates all actuator commands. Operates independently of mission logic.

BInternal State & Logiccore
State machine4 states
OK · WARNING · ESTOP · CRITICAL with precise transitions.
StatesOK motion+tool permitted · WARNING motion still on, degraded · ESTOP motion blocked · CRITICAL battery low, limited motion EvaluationEvery tick. Any trigger transitions toward worse. Recovery needs ALL clear + dwell. DocumentSee SM-SAF-001.
Staleness trackerfail-safe
Tracks last-received timestamp per subscribed topic.
RuleStale → worst-case. Absence of data is unsafe, not safe. Implementation10 Hz timer walks the table.
Decel ramp generatorsmoothing
Controlled velocity ramp when transitioning to stop.
Formulav(t) = sign(v₀) × max(0, |v₀| − a × Δt), a = decel_rate_mps2 ExceptionTilt-triggered ESTOP bypasses ramp — rover already unstable.
Latch managerpersistence
Distinguishes auto-clearable vs. latched ESTOPs.
Auto-clearableperson-in-range, tilt, battery warning. Latchede-stop button, lift (iter 2+), explicit remote e-stop, rogue publisher. Require /safety/reset.
Callback groupsthreading
Reentrant groups + MultiThreadedExecutor.
WhySafety-critical callbacks must not block each other. Default group is mutually exclusive — unsafe. PatternOne reentrant group for subscriptions, mutex-protected shared state read in a timer callback.
