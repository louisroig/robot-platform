boundary_ingester

Persists user-drawn zone polygons and serves the currently-active one.

1.0

Purpose

Accepts polygons from the app (via mission_api), validates them, persists them to disk, and serves the active zone on /mapping/current_zone. Provides services to list, load, activate, and delete zones.

2.0

Scope

In: polygon validation, persistence, lat/lon → map-frame conversion, active-zone publication. Out: drawing UI (mobile app); georeferencing (georeferencer); coverage planning (iteration 2).

3.0

Interfaces

Service servers: /mapping/load_zone (activate by name), /mapping/list_zones. Publisher: /mapping/current_zone, /diagnostics. Internal: disk persistence at /var/lib/platform/zones/.

4.0

Functional

  • SRS-MAP-004-F01
    On POST /zones request (via mission_api), validate the polygon per §6 and persist to zones/<name>.yaml.
    verified by: TEST-MAP-003
  • SRS-MAP-004-F02
    Publish the activated zone on /mapping/current_zone with TRANSIENT_LOCAL durability; immediately deliver to late subscribers.
    verified by: TEST-MAP-001
  • SRS-MAP-004-F03
    Convert polygon vertices from lat/lon to map-frame meters using the active aerial-image origin before publication.
    verified by: TEST-MAP-002
5.0

Performance

Zone load (activate) latency ≤ 500 ms including disk read and publication.

6.0

Validation Rules

  • Outer polygon: ≥ 3 vertices; simple (non-self-intersecting); counter-clockwise oriented; minimum area 4 m².
  • Exclusion polygons: simple; fully contained within outer; non-overlapping with each other.
  • Vertex coordinates within ± 90° lat, ± 180° lon; within 500 m of aerial-image center (rejects typos).
7.0

Dependencies

rclpy, Shapely, pyproj, platform_msgs.

8.0

Configuration

9.0

Failure Modes

Disk full: persistence fails; service returns error. Invalid polygon: rejected before persistence. No zones on startup: publish nothing (geofence_enforcer treats as no-zone mode).
