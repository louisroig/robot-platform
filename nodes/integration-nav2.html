Nav2 Configuration

Iteration-2 deployment notes for the Navigation 2 stack. In iteration 1 Nav2 is present but not invoked; its integration shape is defined here so that iteration-2 work is wiring rather than design.

Purpose

Nav2 is the ROS 2 navigation stack. It provides a planner, local controller, recovery behaviors, costmap stack, and a behavior tree. Iteration 2 introduces autonomous waypoint navigation; this document describes how we'll wire it in.

2.0

Selected Algorithms

ComponentChoiceWhy
Global plannerSmacPlanner2DGood with static costmaps; supports differential-drive constraints.
Local controllerRegulatedPurePursuitControllerRobust, easy to tune for skid-steer; handles slow speeds.
Recovery behaviorsSpin, BackUp, Wait (default set)Safety-compatible — each is velocity-bounded.
Costmap inflationDefault with obstacle layerIteration 2 introduces /perception/obstacles as input.
3.0

Topic Mapping

Nav2 outputs /cmd_vel; we remap to /hal/cmd_vel_auto so that all motion still flows through safety_monitor. Nav2 never bypasses the safety gate.

4.0

Safety Compatibility

  • Nav2 velocity is clipped by safety_monitor just like teleop.
  • Nav2's own recovery behaviors are additional to safety_monitor's ESTOP — they handle navigation-scope faults (stuck, no plan found), not safety faults.
  • On ESTOP, Nav2 receives /hal/cmd_vel_safe=0 (consumed by its controller as "the robot isn't moving"); it will wait or trigger its own recovery.
