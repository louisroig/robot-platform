georeferencer

Computes geographic corner coordinates for captured aerial imagery from drone pose and camera model.

Document ID
SRS-MAP-003
Revision
0.1 · draft
Owner
platform_mapping
Language
Python (rclpy + pyproj)
Classification
Data processing
Last Updated
2026-04-19
1.0

Purpose

Given a captured image + drone pose (lat/lon/altitude/attitude at shutter), compute the four geographic corners of the image on a flat-ground assumption. Publish as /mapping/latest_aerial_image.

2.0

Scope

In: flat-ground pinhole-camera projection; WGS84 geodetic math; AerialImageMeta publication. Out: DEM-aware georeferencing (iter 2); orthorectification; image warping.

3.0

Interfaces

Service server: /mapping/georeference (takes image_path + drone pose, returns AerialImageMeta). Publisher: /mapping/latest_aerial_image, /diagnostics.

4.0

Functional

  • SRS-MAP-003-F01
    Compute corner lat/lon using pinhole-camera footprint at the altitude above ground, rotated by capture attitude.
    verified by: TEST-MAP-011
  • SRS-MAP-003-F02
    Publish AerialImageMeta immediately after successful computation; include image_path reference.
    verified by: TEST-MAP-010
5.0

Performance

Per-image processing < 500 ms.

6.0

Safety

Not safety-critical. Output used to draw boundaries; operator eyes-on acceptance is the enforcement step.

7.0

Dependencies

rclpy, pyproj (for lat/lon math), numpy, platform_msgs.

8.0

Configuration

ParameterDefault
camera_hfov_deg62.2 (OV2640 default)
camera_vfov_deg48.8
ground_modelflat
9.0

Failure Modes

Missing pose data: service returns failure. Nonsense altitude (< 5 m or > 100 m): log warning and still compute (operator decides). Image file missing: return failure.

10.0

Verification

TEST-MAP-010 (publication timing), TEST-MAP-011 (accuracy with surveyed ground markers).

11.0

Open Issues

  1. OPEN-01 — DEM-aware processing for sloped yards, iteration 2.
  2. OPEN-02 — Camera intrinsic calibration currently static (datasheet values); per-unit calibration for better accuracy.
