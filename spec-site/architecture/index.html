Orientation

Architecture documents set the stage for the rest of the corpus. They explain what the system is, where it lives, how the layers stack up, how a single mission flows end-to-end, which code runs where, and — the discipline document — what belongs in the platform versus the application.

These documents are the contract between the whole-system view (who this is for, what it does) and the component view (individual nodes and interfaces). Readers coming from a requirement or an ICD will often surface back here to find the larger picture. Readers starting from the splash page typically read these first, in order.

Reading order

If you're new to the project: read System Context first, then Layered Architecture, then Mission Data Flow. Read Platform vs Application before you contribute code. Deployment and Iteration Roadmap can be referenced as needed.
