Magnetic field vector from the MMC5983MA magnetometer. Used for yaw-drift correction in the global EKF.

Deferred to iteration 2

The MMC5983MA is not fitted in iteration 1 per HW-PI5-001 §6 and SRS-HAL-002 §11 OPEN-02. /hal/imu/mag is therefore not published during iterations M1–M6. This ICD is retained as a reference so that the iteration-2 reintroduction does not require re-drafting. Sections below describe the intended behavior at the time of reintroduction — they are not in force during iteration 1.

1.0

Purpose

Provides Earth-frame magnetic field readings for yaw-drift correction in the global EKF. Independent of accel+gyro; publishes at a lower rate because magnetic field changes slowly.

Not used for safety. Optional input; if magnetometer fails, EKF continues with degraded yaw accuracy (drift unbounded over long durations).

3.0

Message

Type: sensor_msgs/msg/MagneticField. Field magnetic_field populated in Tesla. Covariance populated from calibration.

4.0

Semantics

Hard-iron and soft-iron calibration is applied by the driver. Raw sensor output is transformed by a calibration matrix stored in mower_config/imu_calibration.yaml. Calibration is re-done per-yard (large metal structures shift the null point).

Values represent Earth's magnetic field at the rover's location in the imu_link frame. Typical magnitude ~50 µT; declination depends on location.

5.0

Timing

50 Hz ± 5 Hz. Read latency ≤ 8 ms over I²C.

6.0

QoS

RELIABLE, KEEP_LAST depth 5, VOLATILE. No deadline — EKF tolerates short gaps.

7.0

Fault

FaultResponse
Mag I²C faultDriver suspends mag publication, continues IMU. Emits diagnostic.
Unexpectedly large field magnitude (> 200 µT)Likely local magnetic disturbance; driver emits warning. EKF down-weights.
Calibration file missingDriver refuses to publish mag; IMU continues; EKF operates without yaw correction (degraded).
