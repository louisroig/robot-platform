Fused pose estimate in the map frame. The authoritative "where is the rover" for mission-level code.

1.0

Purpose

Output of the global EKF. Fuses local odometry (/odometry/filtered), SLAM pose (RTAB-Map), and optional GPS into a single best-estimate pose in the map frame. This is the pose the geofence_enforcer checks against the active zone; it is the pose the mobile app shows on the aerial image.

3.0

Message

Type: geometry_msgs/msg/PoseWithCovarianceStamped. Standard fields: pose (position + orientation quaternion) in map frame; 6×6 covariance matrix reflecting EKF uncertainty.

Frame id: map. Pose is expressed in meters (x, y) and radians (yaw), with z typically pinned at 0 for the ground rover.

4.0

Semantics

The map frame is anchored to the first SLAM keyframe in the current session. Persistence across sessions is not yet implemented (iteration 2+). Consumers should not assume map frame is consistent day-over-day.

On startup, before SLAM converges, pose is published with high covariance (identity × 100) and a header stamp. Consumers should check the pose.covariance[0] element (x-variance) and down-weight or reject when above their tolerance.

5.0

Timing

30 Hz ± 3 Hz. Rate set by EKF's update cycle.

6.0

QoS

RELIABLE, KEEP_LAST depth 5, VOLATILE.

7.0

Fault

FaultResponse
SLAM loss of trackingCovariance grows rapidly; ekf_global continues publishing using odometry dead-reckoning. Consumers detect via covariance threshold.
ekf_global crashTopic goes silent; geofence_enforcer ceases publishing; safety_monitor staleness triggers ESTOP.
SLAM loop-closure jumpDiscontinuity in pose. Detected by consumers as position jump > 0.5 m per sample; geofence responds with OUTSIDE.
