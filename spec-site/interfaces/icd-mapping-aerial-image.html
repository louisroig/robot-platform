/mapping/latest_aerial_image

Georeferenced aerial image metadata from the most recent drone mapping flight. The image binary is served via REST; this topic carries the metadata.

Purpose

Produced by georeferencer after a mapping mission completes. Describes the captured aerial image: file path on rover disk, capture pose, and the four georeferenced image-corner coordinates. Consumed by mission_api when the mobile app requests mission results.

4.0

Semantics

Georeferencing assumes a flat-ground model at the elevation of the takeoff point. For yard-scale captures this is a good approximation (errors < 0.5 m at yard corners for a 30 m altitude).

The image_path is server-local; the mobile app does not fetch from this path. It fetches via a REST endpoint (GET /missions/{id}/image) exposed by mission_api, which resolves the path internally.

5.0

Timing

Published on-complete (one message per successful mapping mission). No regular cadence.
