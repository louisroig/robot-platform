Glossary

Alphabetical index of acronyms and terms used across the corpus.

Document ID
GLOSSARY
Revision
0.1 · draft
Last Updated
2026-04-19
A
  • AIOAll-In-One — flight controller with an integrated ESC. The drone uses an iFlight Blitz Whoop F7 AIO.
  • ArduCopterOpen-source autopilot firmware for multirotors. Running on the Blitz F7 FC in iteration 1.
B
  • BDDBlock Definition Diagram — a SysML structural diagram.
  • BECBattery Eliminator Circuit — voltage regulator that steps down the main battery to a logic-level rail.
C
  • CSEPCertified Systems Engineering Professional — INCOSE credential.
D
  • DDSData Distribution Service — the pub/sub middleware that underlies ROS 2 transport.
E
  • EKFExtended Kalman Filter — the nonlinear state estimator used by robot_localization.
  • ELRSExpressLRS — open-source long-range RC link. Used for drone manual-override control.
  • ESCElectronic Speed Controller — drives the drone's motors.
  • ESTOPEmergency stop. A safety state that latches motion off until cleared.
F
  • FCFlight Controller — runs the autopilot firmware on the drone.
  • FSOFacility Security Officer — cleared-industry role. Mentioned for context; not a project role.
G
  • GCSGround Control Station — desktop/tablet software for commanding and monitoring the drone (e.g., QGroundControl).
  • GeofenceA virtual boundary. In this system, the user-drawn polygon outside of which the rover must not travel.
H
  • HALHardware Abstraction Layer — the vendor-boundary layer between vendor-specific drivers and vendor-neutral platform code.
I
  • ICDInterface Control Document — the authoritative specification for one topic, service, action, or external API.
  • IMUInertial Measurement Unit — accelerometer + gyroscope, typically also a magnetometer.
  • INCOSEInternational Council on Systems Engineering.
  • IRADInternal Research And Development — company-funded R&D (used only in background context).
L
  • LAANCLow Altitude Authorization and Notification Capability — FAA airspace-authorization system for UAS.
  • LiFePO4Lithium Iron Phosphate — a safer, lower-energy-density battery chemistry used on the rover.
M
  • MBSEModel-Based Systems Engineering — the discipline this corpus takes its language from, intentionally applied in HTML rather than in a tool like Cameo.
  • MCUMicrocontroller Unit — a small embedded processor. Used for the future independent safety board.
N
  • NCNormally Closed — switch convention. The physical e-stop is NC so that a broken wire fails safe.
  • NPUNeural Processing Unit — the dedicated YOLO inference accelerator onboard the OAK-D Lite FF.
O
  • OCSMPOMG Certified Systems Modeling Professional — the SysML credential.
Q
  • QGCQGroundControl — the ArduPilot/PX4 ground station application.
  • QoSQuality of Service — DDS transport parameters: reliability, durability, history, depth, deadline, lifespan.
R
  • ROS 2Robot Operating System v2 — the middleware and tooling the platform is built on. Target distribution: Jazzy.
  • RTRemote Terminal — MIL-STD-1553 term, used here as an analogy for the single-authority bus-master pattern of safety_monitor.
  • RTKReal-Time Kinematic — centimeter-level GPS correction. Future iteration; optional in iteration 1.
S
  • SiKFirmware name for the common 3DR-style telemetry radios. Not used in iteration 1.
  • SLAMSimultaneous Localization And Mapping — building a map while localizing against it.
  • SRSSoftware Requirements Specification — per-component document listing what the component shall do.
  • SysMLSystems Modeling Language — the OMG standard typically rendered in tools like Cameo.
T
  • TFTransform — the ROS coordinate-frame tree. Maintained by robot_state_publisher and the EKF nodes.
U
  • ULUnderwriters Laboratories — the safety certification body relevant for commercial outdoor robotics.
  • URDFUnified Robot Description Format — XML description of robot geometry and kinematics.
V
  • VTXVideo Transmitter — analog/digital FPV video link. Not used on the mapping drone in iteration 1.
