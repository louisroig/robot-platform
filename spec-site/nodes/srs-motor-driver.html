motor_driver

Open-loop PWM control of two BTS7960 H-bridge modules for track drive. Converts Twist commands to per-track PWM duty cycles and direction signals. No encoder feedback; no battery telemetry.

1.0

Purpose

The motor_driver is the vendor-boundary node for two BTS7960 43A H-bridge modules, controlling the left and right aluminum tracks. Upstream it exposes ROS-standard Twist commands; downstream it drives GPIO-based PWM (speed) and direction signals to each H-bridge. This is open-loop control: no encoder feedback. Above it, the system treats motor control as generic; below it, the driver handles all GPIO-level specifics.

2.0

Scope

In scope. GPIO PWM + direction output to the BTS7960 pair; Twist-to-per-track velocity kinematics; command-timeout safe-halt; basic thermal shutdown detection via stalled-motion cross-check with IMU.

Out of scope. Encoder reading and odometry (deferred to M5 EKF). Battery voltage monitoring (deferred, requires external ADC). Nav2 costmap, trajectory following, slip compensation.

Functional Requirements

  • SRS-HAL-001-F01
    Convert every received Twist into per-track velocities via skid-steer kinematics using configured track width.
    verified by: TEST-HAL-001
  • SRS-HAL-001-F02
    Clip commanded velocities to configured max_linear_vel and max_angular_vel with sign preservation.
    verified by: TEST-HAL-012
  • SRS-HAL-001-F03
    Drive BTS7960 PWM pins at configured frequency (default 2 kHz) with duty cycle 0–100% and direction pins set per kinematics output.
    verified by: TEST-HAL-002
  • SRS-HAL-001-F04
    Halt motor commands if no /hal/cmd_vel_safe message has been received within cmd_vel_timeout_ms (default 500 ms).
    verified by: TEST-HAL-009
  • SRS-HAL-001-F05
    Reject Twist messages containing NaN or Inf in any field; emit diagnostic; hold last-known-good command or halt per policy.
    verified by: TEST-HAL-011
5.0

Performance Requirements

  • SRS-HAL-001-P01
    GPIO write cycle per control frame ≤ 5 ms (PWM + DIR × 4 pins).
    verified by: TEST-HAL-013
  • SRS-HAL-001-P02
    CPU usage < 3% on Pi 5.
    verified by: TEST-HAL-014
6.0

Safety Requirements

  • SRS-HAL-001-S01
    Command timeout (F04) is the subscriber-side component of the two-layer safety: even if safety_monitor fails to publish zero, stale commands themselves cause halt.
    maps to: SR-005
  • SRS-HAL-001-S02
    On startup, motors shall be held at zero velocity until first valid /hal/cmd_vel_safe is received.
    maps to: SR-008
