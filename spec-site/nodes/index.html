About this section

Every custom ROS 2 node has a Software Requirements Specification here. Each SRS covers purpose, interfaces (with cross-links to the corresponding ICDs), functional and performance requirements, dependencies, configuration parameters, failure modes, and verification. Integration notes for off-the-shelf nodes are brief — the configuration parameters and rationale are captured; the full ROS 2 API is not re-documented.

Anatomy diagrams use a mind-map-style visualization for nodes whose internal structure is non-obvious from prose alone. They are complementary to the SRS, not a replacement for it.
