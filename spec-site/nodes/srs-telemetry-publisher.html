telemetry_publisher

Serves the live-telemetry WebSocket to the mobile app at 5 Hz. Aggregates pose, safety, battery, and mission state into a single JSON stream.

1.0

Purpose

Serves wss://rover.local:8443/ws/telemetry, pushing the JSON payload defined in ICD-API-002 at 5 Hz. Also pushes an immediate message on any safety-state change (burst mode) so the operator sees state transitions without waiting for the next tick.

2.0

Scope

In: WebSocket server, auth at handshake, aggregation of topic latest-values, 5 Hz push, burst on state change. Out: request-response endpoints (mission_api's job).

3.0

Interfaces

External: wss on TCP 8443 at /ws/telemetry. ROS 2 subscribers: /safety/state, /localization/pose, /hal/battery, /perception/people, /perception/obstacles, /geofence/state.

4.0

Functional

  • SRS-API-002-F01
    Emit a JSON payload conforming to ICD-API-002 §4 at 5 Hz.
    verified by: TEST-API-010
  • SRS-API-002-F02
    On safety state change, push a burst message within 100 ms.
    verified by: TEST-API-011
  • SRS-API-002-F03
    Reject handshakes without a valid bearer token with HTTP 401.
    verified by: TEST-API-012
  • SRS-API-002-F04
    Support at least 4 concurrent client connections.
    verified by: TEST-API-013
5.0

Performance

CPU < 5% with single client. Per-message serialization < 5 ms.

6.0

Safety

Not safety-critical. Failure causes app to lose visibility but does not affect rover behavior.

7.0

Dependencies

rclpy, websockets, shared token store with mission_api.

9.0
